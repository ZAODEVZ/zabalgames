## Day 6 - our first fireside

To close the week I sat down with Carlos (Plat0x), the developer behind Bonfires, and it turned into half working session, half vibe-coding masterclass. The lessons, if you are on the builder track:

- Plan, then goal. Give your agent a plan plus one goal with a measurable threshold (say, under five seconds of query latency) so it keeps working until it hits the number.
- Diagram Bob and Alice before you write a line of code. Define the exchange and the sequence first, language last.
- Prompt types first, and use a protocol or adapter contract so you can swap the database later without a rewrite.
- Documentation as code: strip out all the code and the docs alone should rebuild it one to one.

Watch the fireside: https://zabalgamez.com/recordings/fireside/1
