# FounderCheck Reframe - Session 2

> Date: 2026-05-29. Response to the FounderCheck export (viability 3.35, gate BLOCK).
> TL;DR: the tool scored the wrong product. The BLOCK is a category error, not a
> signal to stop. Below: why, what is still useful, and how to re-run it correctly.

## What FounderCheck evaluated

It profiled the customer as "indie game developers and hobbyists who use vibe coding
tools to build games" and the problem as "struggling to distribute their vibe-coded
games and share knowledge." That is a **SaaS for vibe-coder game distribution +
knowledge sharing**.

That is not ZABAL Gamez. ZABAL Gamez is The ZAO's 3-month Build-A-Thon - a
creator-onboarding event (see `docs/positioning-2026-05-29.md`). Grading it as a
subscription tool is like grading a music festival on its app-store rating.

The most useful thing in the report is the error itself: an AI fed the materials
concluded "game distribution tool for vibe coders." If a model reads it that way, so
will people. That is a brand-clarity problem, addressed in the positioning doc.

## Score-by-score reframe

| FounderCheck score | What it assumed | ZABAL Gamez reality | Right metric |
|---|---|---|---|
| pain_severity 3 | vibe coders can't distribute games | The job is onboarding builders into The ZAO and getting tools built on its rails - not solving game distribution | Builder demand for context + mentorship + a community to build for |
| customer_clarity 7 | a payer persona | Audience is real and reachable (Farcaster builders), but they are participants, not subscribers | Number of qualified builders reached |
| channel_accessibility 5 | acquisition channels for a tool | Farcaster / X / FarHack / the Agentic Bootcamp are warm, owned-adjacent channels | Reply + signup rate from those channels |
| conversion_friction 2 | signup-to-paid funnel | There is no paywall; "conversion" is build-and-ship | Submissions that hit the bar (live URL + repo + 60s demo + /zabal cast) |
| willingness_to_pay $0-15/mo | a subscription | Builders are paid (pool + WaveWarZ cut + NFT + Respect); The ZAO funds it | Cost per onboarded builder vs ecosystem value created |

## What the report got right (keep it, retarget it)

- The audience lives on Farcaster, X, and Build-A-Thon scenes. True - use it.
- Pick ONE problem and say it plainly. True - the brand's clarity test. For the event
  the one problem is: "talented Farcaster builders have nowhere to build *for* an
  ecosystem and earn a stake in it." Lead with that.
- Talk to real humans before scaling. True - but the right humans are Farcaster
  builders (FarHack / bootcamp alumni), NOT r/indiegaming. The suggested
  r/indiegaming experiment is aimed at the wrong crowd.
- Run a manual MVP. True - a small July cohort with hands-on mentor pairing is exactly
  that. If ~20 builders are not genuinely interested, tighten the offer first.

## If you want to re-run FounderCheck

Feed it this description so it scores the right thing:

> ZABAL Gamez is a 3-month Build-A-Thon run by The ZAO, a 100+ member Farcaster music
> and creator ecosystem. It onboards Farcaster builders by giving them deep ecosystem
> context, a mentor as an embedded teammate, a shared $500 USDC prize pool plus ongoing
> on-chain economics, governance (Respect), and distribution - in exchange for shipping
> real open-source tools on The ZAO's rails. It is an event and an onboarding funnel,
> not a SaaS. Success is measured by builders onboarded, tools shipped, and ecosystem
> value, not subscription revenue.

Expect the gate logic to still look for a payer and a paid funnel - which is why the
right scorecard for an event is the metrics in the table above, not a viability score.

## The one legitimately open question

There may be a *separate* product hiding here - for example, a surface where
ZAO/Farcaster vibe-coded builds get discovered and distributed. If that interests you,
validate it on its own terms (its own customer, its own one problem, real distribution
pain from real builders) and do not conflate it with the event brand. The event does
not need it to be valid; it is already a coherent onboarding program.

## Success metrics to track instead of a viability score

- Builders signed up per channel, and reply rate to direct outreach.
- July submissions that hit the bar (live URL + open repo + 60s demo + /zabal cast).
- Mentor pairings made.
- Respect distributed to active participants.
- Repos still live + maintained 30 days after the Finals.
- ZAO rails wired per build (Empire, Hats, Bonfire, EAS, Farcaster) - more rails, more
  ecosystem value.
