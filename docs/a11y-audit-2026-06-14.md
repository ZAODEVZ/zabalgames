# Accessibility audit - 2026-06-14 (non-recording surfaces)

> The follow-up a11y pass flagged in `docs/code-hygiene-audit-2026-06-14.md`. Scope: the site's
> non-recording HTML pages. Recording surfaces were excluded per the active two-session split
> (`recordings/*.html`, `recaps.json`, `data/streams/**/transcripts/*`, `recordings/index.json`).
> Method: scripted static scan of the 33 top-level `*.html` pages for the high-signal,
> machine-checkable a11y basics. Brand: no emojis, no em dashes.

## Result: clean on the static checks. No fixes needed.

Checks run across all 33 pages:

| Check | Result |
|---|---|
| `<html lang=...>` present | All 33 pages set it. None missing. |
| Images with an `alt` attribute | All real `<img>` elements have `alt`. (See the one false positive below.) |
| Single `<h1>` per page (no multiple/skipped top headings) | No page has more than one `<h1>`. |
| Form controls labelled (`id` / `aria-label` / `aria-labelledby`) | No unlabelled `input` / `select` / `textarea` found. |

### The one flag, and why it is a false positive
The scan flagged `p.html` for an `<img>` without `alt`. That match is a **code comment** -
`// Async-hydrate an <img> with the HAATZ pfp once it arrives` (p.html:363), not a real element.
The actual avatar images p.html builds in its JS template strings both set `alt` (p.html:451 and
p.html:481, `alt="' + escapeHtml(p.name) + '"`). No change made.

## What this pass did NOT cover (needs a browser, future work)

The static scan catches structural basics only. A deeper pass should use a headless browser /
axe-core and check:
- Color contrast against the dark theme (text vs `--text-muted` / `--text-dim` on `--surface`).
- Visible focus states + keyboard focus order on interactive elements (the chip filters, chapter
  buttons, share buttons, nav).
- ARIA roles/states on custom widgets (the comment widget, the schedule filter, the leaderboard).
- Dynamically injected content having accessible names once hydrated (avatars, counts).
- `prefers-reduced-motion` handling for any animation.
- Screen-reader smoke test of the primary flows (home -> schedule -> a recording).
