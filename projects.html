Path 1 - Adopt a Project

Started ZAO Projects Up for Grabs.

You do not have to build from scratch. These are real, in-progress ZAO ecosystem projects that need a builder to pick them up and ship them forward. Adopt one in your July submission, OR build from scratch with the context file. Both paths count.

How adoption works

Pick a project. DM @bettercallzaal with the project number to claim it (first-come-first-served, one builder per project). You ship it for ZABAL Games Season 1 - keep the repo + the credit. The project stays open-source MIT.

Status updates here as builders claim slots. Full curated list also in llms.txt for any AI harness to load.

The eight

01

Songjam leaderboard migration

Move the Songjam leaderboard off the deprecated X scraper onto an Empire Builder API leaderboard. Frees Adam from scraper maintenance, gives ZABAL Farcaster-native data. Pattern is the standard apiLeaderboards JSON contract: publish [{address, score}], Empire Builder pulls + distributes.

Spec: Doc 654 - Empire V3 meeting decisions
02

POIDH bounty leaderboard on Empire Builder

Empire Builder leaderboard pulling on-chain POIDH bounty submissions. Counts unique submitters across all ZABAL bounties (5 submissions on one bounty counts as 1; submitters across two bounties count as 2). One contract is wired; second pending. Replaces manual $1-ZABAL-per-submitter outreach.

Spec: Doc 654 + the Player Context Bundle for the apiLeaderboards JSON contract
03

zlank.online Empire daily-stats Snap template

One-click "Today's Empire Builder Stats" Snap template on zlank.online. Footy app is the reference. kmac.eth + Zaal collab. The Snap pulls the day's leaderboard movement, formats a cast, returns a button that posts it. Reusable for any empire (POIDH, Songjam, etc).

Spec: Doc 654
04

Twitch -> Empire Builder stream feed

StreamElements WebSocket events feed an apiLeaderboard endpoint. Stream activity (tips, raids, follows, chat engagement) becomes a real-time Empire leaderboard surface. Generalizes to any creator running Twitch + an empire.

Spec: pull the Twitch + StreamElements sections from the player context bundle
05

A new ZOE skill

Extend the ZOE concierge bot (Telegram @zaoclaw_bot). Backend mirrors the Hermes runtime pattern - Claude Code CLI as subprocess, swap the system prompt for the new skill's persona. Skills are independent capabilities ZOE can route to.

Spec: ZOE / Hermes references in the player context bundle. Pick a skill the ZAO actually needs.
06

This /zabal mini app

The Farcaster mini app you're on. Extend the workshops calendar, the submissions board, the connector NFT flow. Repo is open: github.com/ZAODEVZ/zabalgames. Highest meta-leverage build - you literally extend the platform you're shipping into.

Spec: TODO.md + docs/snap-design.md
07

COC Concertz content pipeline

Today: record show -> Descript edit -> newsletter builder generates YouTube descriptions (MiniMax AI) -> manual cross-post to YouTube + social. Automate the cross-post end. Bridges to ZAO's src/lib/publish/ 10-platform module. Recurring win - every COC concert.

Spec: COC Concertz references in the player context bundle
08

Streaming auto-clip flywheel

Turn 24h of stream into ~150 short clips for TikTok / Reels / YouTube / Farcaster. The content engine that processes ZABAL Games streams (and every future ZAO event). High lift, high payoff - it would auto-amplify the whole Season.

Spec: Doc 701 Part 4 + the player context bundle

Or build from scratch

Path 2 is wide open. Build anything that helps the ZAO community using the rails the ecosystem already has (Empire Builder, Farcaster mini apps, Hats, Bonfire, EAS, Hypersub, Coinflow). Composability is the point - do not reinvent identity, leaderboards, or attestations.

Load zabalgames.com/llms.txt into your AI harness and you have the full ecosystem context. Look for the gap that wants to be filled.
